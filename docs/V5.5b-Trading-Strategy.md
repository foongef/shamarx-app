## Strategy Architecture

The system runs on M15 candles with H1 bias overlay. Each candle tick:

1. Compute indicators (M15 + H1)
2. Manage open positions (SL/TP/BE/trail)
3. Check risk gates (can we trade?)
4. Generate signals via priority-chain routing
5. Score signal quality (0-100)
6. Size the position (risk-adjusted)
7. Open position(s) — single or locker/runner split

---

## 1. Entry Engines (Priority Chain)

Signals are evaluated in priority order — first match wins:

### Engine A: STRONG_TREND (ADX >= 25)

**Conditions:**
- H1 ADX >= 25 AND ADX rising (current > 2 bars back)
- DI separation: |+DI - -DI| >= 3
- Price pulls back to EMA20/EMA50 zone
- RSI filter: BUY 40-65, SELL 35-60
- Confirmation candle required (engulfing or strong close)

**Targets:**
- SL: Below/above swing point, clamped to ATR x 3.0
- TP: 2.0R (removed at 2.0R favorable for trail)

### Engine B: FVG Fill (ADX >= 15)

**Conditions:**
- H1 ADX >= 15
- Valid Fair Value Gap detected (3-candle pattern)
- Middle candle body >= 40% of range, range >= 0.5x ATR
- Price fills into the gap zone
- FVG window: 96 candles (24 hours)
- ATR hard reject: SL distance > 2.5x ATR = skip

**Targets:**
- SL: Opposite side of FVG
- TP: Based on gap size and structure

### Engine C: Range (ADX < 20, not DEFENSIVE)

**Conditions:**
- H1 ADX < 20
- NOT in DEFENSIVE trading mode
- Price at EMA50 +/- 1.5x ATR bands
- RSI confirmation: BUY < 30, SELL > 70
- Confirmation candle required (engulfing, strong close, or pin bar)

**Targets:**
- SL: Beyond ATR band
- TP: Mean reversion to EMA50

### Disabled Engines

- **BOS_RETEST** — Disabled in V5.4. Net loss across all years.
- **MOMENTUM_CONT** — Disabled in V5.4. Zero TPs, pure losses.

## 2. Quality Score (0-100 points)

Every signal gets scored before entry. The score gates lot sizing and locker/runner split.

| Factor | Condition | Points |
|--------|-----------|--------|
| **H1 ADX Strength** | ADX >= 30 | +20 |
| | ADX 25-29 | +10 |
| | ADX 20-24 | +5 |
| **RSI Alignment** | Mid-range (40-60) for trade direction | +10 |
| **H1 Bias Alignment** | Trade direction matches H1 EMA bias | +15 |
| **Confirmation Quality** | Engulfing pattern | +10 |
| | Strong close or pin bar | +8 |
| **Regime Stability** | Same H1 regime for 3+ bars (3 hours) | +10 |
| | Same H1 regime for 2 bars | +5 |
| **EMA Stack Alignment** | Perfect order (e.g., EMA20 > EMA50 > EMA200 for BUY) | +10 |
| **Session Quality** | London-NY overlap (13:00-16:00 London / 08:00-11:00 NY) | +7 |
| | London session | +5 |
| | NY session | +3 |
| **Volatility Health** | ATR/ATRBaseline between 0.8 and 1.3 | +10 |

**Quality tiers and their effects:**

| Score | Tier | Effect |
|-------|------|--------|
| >= 65 | High-Q | Locker/runner split (1.3x exposure) |
| 40-64 | Mid-Q | Normal lot size (1.0x) |
| < 40 | Low-Q | Lot size dampened (0.6x) |

## 3. Lot Sizing Formula

```
effectiveRisk = baseRiskPercent

// Step 1: Drawdown scaling
if overallDD > 20%  → effectiveRisk *= 0.3
else if overallDD > 10% → effectiveRisk *= 0.5

// Step 2: DEFENSIVE mode (with recovery gate)
if DEFENSIVE mode AND consecutiveWins < 3 → effectiveRisk *= 0.5

// Step 3: Quality dampener
if qualityScore < 40 → effectiveRisk *= 0.6

// Final calculation
riskAmount = balance * (effectiveRisk / 100)
lotSize = riskAmount / (slPoints * lotSizeUnits)
lotSize = clamp(lotSize, 0.01, 1.0)
```

**Example ($1,000 balance, 3% risk, 10-point SL, NORMAL mode, Q=50):**
```
effectiveRisk = 3.0%
riskAmount = $1,000 * 0.03 = $30
lotSize = $30 / (10 * 100) = 0.03
```

**Same trade in DEFENSIVE mode with Q=35:**
```
effectiveRisk = 3.0% * 0.5 * 0.6 = 0.9%
riskAmount = $1,000 * 0.009 = $9
lotSize = $9 / (10 * 100) = 0.01 (floor)
```

## 4. Position Management

### Breakeven Activation

| Setup Type | Favorable Move Required | BE SL Placement |
|------------|------------------------|-----------------|
| TREND_PULLBACK | 1.0R | Entry + 0.1R (10% buffer) |
| FVG Fill | 0.8R | Entry + 0.1R (10% buffer) |

Once BE activates, the position is risk-free. The trailing system takes over.

### Trailing Stop (after BE activation)

**TREND_PULLBACK — 5-tier aggressive trail:**

| Favorable Move | Trail Distance from Peak |
|---------------|------------------------|
| >= 5.0R | 0.4R (very tight) |
| >= 4.0R | 0.5R |
| >= 3.0R | 0.6R |
| >= 2.0R | 0.75R |
| < 2.0R | 1.0R (at breakeven) |

At >= 2.0R favorable: TP is removed, letting the trail capture the full trend move.

**FVG Fill — 4-tier wider trail:**

| Favorable Move | Trail Distance from Peak |
|---------------|------------------------|
| >= 3.0R | 0.5R |
| >= 2.0R | 0.6R |
| >= 1.5R | 0.7R |
| < 1.5R | 1.0R (at breakeven) |

At >= 1.5R favorable: TP is removed.

### Locker/Runner Split (V5.5b)

**Triggered when:** `(STRONG_TREND OR qualityScore >= 65) AND NOT pyramid AND lotSize >= 0.02`

| Position | Lot Size | TP | Purpose |
|----------|----------|-----|---------|
| **Runner** | 100% of calculated lot | Normal TP + trail | Captures the full trend move |
| **Bonus Locker** | 30% of calculated lot | 1.0R | Quick secured profit |

**Total exposure: 1.3x** on high-quality setups.

The runner behaves exactly like a standard single position (full lot, normal TP, normal trailing). The locker is a bonus position that takes a quick profit at 1.0R. This means you don't sacrifice any trend-capture potential — you just add extra wins on top.

## 5. Trading Modes (Adaptive)

Evaluated each candle via `getTradingMode()`:

### DEFENSIVE

**Triggers (any one):**
- Equity drawdown >= 15%
- Consecutive losses >= 4
- Rolling 7-day losses >= 5

**Effects:**
- Lot size halved (unless recovering: 3+ consecutive wins skip halving)
- Range engine disabled
- Pyramiding blocked
- Direction stacking blocked

### AGGRESSIVE

**Triggers (all required):**
- Equity drawdown < 5%
- Consecutive wins >= 2

**Effects:**
- Full risk allocation
- All engines enabled
- Pyramiding allowed

### NORMAL

**Default mode** when neither DEFENSIVE nor AGGRESSIVE conditions are met.

## 6. Risk Management — Circuit Breakers

### Daily Loss Limit
- If `dailyPnL <= -maxDailyLossPercent` of balance → stop trading for the day

### Consecutive Loss Pause (Escalating)

| Consecutive Losses | Pause Duration |
|-------------------|----------------|
| 3 (base threshold) | 1 trading day |
| 4 | 3 trading days |
| 5+ | 5 trading days |

Counter resets only on a TP win or breakeven exit (not on trailing SL profit).

### Rolling 7-Day Loss Stop
- Track all losing trades in a 7-calendar-day window
- >= 4 losses in the window → 5 trading day pause

### Weekly Drawdown Circuit Breaker
- Track peak equity each ISO week
- Weekly DD >= 8% → 5 trading day pause

### Equity Drawdown Tiered Breaker

| DD Level | Pause Duration | Notes |
|----------|---------------|-------|
| >= 15% | 3 trading days | DEFENSIVE mode activates |
| >= 25% | 7 trading days | Stacks with DEFENSIVE |
| >= 35% | 30 trading days | Near-shutdown protection |

Tiers are event-triggered (on new trade) and each tier fires only once per drawdown cycle. When equity makes a new peak, the tier tracker resets.

### Recovery Gate (V5.5b)

When in DEFENSIVE mode, the standard behavior halves lot sizes. The recovery gate **skips this halving after 3 consecutive wins**. This prevents the "death spiral" where reduced sizing makes it impossible to recover from drawdowns — each win is too small to dig out of the hole.

## 7. Entry Filters & Guards

### ATR Hard Reject
- FVG entries only: If SL distance > 2.5x ATR → reject the signal
- Prevents entries with oversized stops during volatile spikes

### Volatility Ceiling
- All entries: If ATR/ATRBaseline >= 2.0 → reject (HIGH_VOLATILITY)
- ATRBaseline = 20-period SMA of ATR14

### Session Filter
- **London:** 08:00-16:59 local (DST-aware)
- **New York:** 08:00-13:59 local (DST-aware)
- Outside these windows: no entries

### Same-Day Direction Limit
- Track SL hits per direction (BUY/SELL) per day
- >= 3 SL hits in same direction on same day → block that direction
- Prevents revenge-trading in one direction

### Cooldown After Exits
- After TP: 1 candle (15 min)
- After breakeven: 1 candle (15 min)
- After SL: 2 candles (30 min)

### Direction Stacking Guards

When adding to an existing direction:

**Guard A — Position State:**
- Block if >= 2 same-direction positions already open
- Block if existing positions NOT at breakeven
- Block if new setup NOT STRONG_TREND
- Block if DEFENSIVE mode
- Block if volatility elevated (ATR/baseline >= 1.3)

**Guard B — Risk Cap:**
- Maximum total same-direction risk: 20% of equity
- Sum up `slDistance * lotSize * lotSizeUnits` for all same-direction positions
- If adding the new position exceeds cap → block

### Pyramid Safety Gate
Pyramiding only allowed when:
- NOT in DEFENSIVE mode
- Equity DD < 10%
- Consecutive losses < 2

## 9. Trade Recording & Exit Classification

| Exit Reason | Consecutive Loss Effect | Consecutive Win Effect |
|-------------|------------------------|----------------------|
| **SL** (loss) | +1 consecutive loss, reset wins | Reset wins to 0 |
| **TP** (win) | Reset losses to 0 | +1 consecutive win |
| **BREAKEVEN** | Reset losses to 0 | No change |
| **Trailed SL** (profit) | No change | No change |

This distinction matters because:
- Only SL losses escalate the pause system
- Breakeven exits are treated as "reached profit threshold" — they reset the loss streak
- Trailed SL profits don't affect either counter (conservative approach)

## 10. V5.5b Specific Changes (vs V5.5 Baseline)

| Feature | V5.5 Baseline | V5.5b |
|---------|--------------|-------|
| DEFENSIVE lot halving | Always halved | **Skip halving after 3 consecutive wins** (recovery gate) |
| Low-Q lot sizing | No adjustment | **0.6x for qualityScore < 40** |
| High-Q position structure | Single position | **Full runner + 30% bonus locker at 1.0R** |
| Locker/runner trigger | STRONG_TREND only | **STRONG_TREND OR qualityScore >= 65** |
| Locker/runner split ratio | 50/50 | **100% runner + 30% bonus locker** |

### Why These Changes Work Together

1. **Recovery gate** prevents the death spiral: after 3 wins during DEFENSIVE, you get full lot sizing back, enabling meaningful equity rebuilding (fixes 2026 drawdown).

2. **Low-Q dampener (0.6x)** counteracts the butterfly effect: the recovery gate changes lot sizes, which cascades through the equity curve and introduces extra trades. Many of these are low-quality setups — the 0.6x sizing limits their damage.

3. **Bonus locker on high-Q** amplifies your best setups without sacrificing trend capture. Unlike a 50/50 split (which halves the runner), the bonus approach keeps the runner at full size and adds a small extra position that locks in a quick 1.0R win. This means big trends still get captured at full lot, plus you pocket a bonus.

## 11. Signal Generation Flow (Pseudocode)

```
FOR each M15 candle:

  // Indicators
  Compute M15: EMA20, EMA50, EMA200, RSI14, ATR14, ADX14
  Compute H1: EMA20, EMA50, ADX14, +DI, -DI (via binary search alignment)

  // Position management
  FOR each open position:
    Check SL hit (with spread adjustment)
    Check TP hit
    Update peak favorable price
    Check breakeven activation (1.0R standard, 0.8R FVG)
    If at BE → apply trailing stop tiers
    Update trailing SL

  // Risk gates
  IF NOT riskManager.canTrade(date, openCount): SKIP
  IF in cooldown: SKIP
  IF volatility ceiling breached (ATR/baseline >= 2.0): SKIP
  IF NOT in active session (London/NY): SKIP

  // Signal generation (priority chain)
  signal = null
  IF H1 ADX >= 25 AND ADX rising:
    signal = tryStrongTrendEntry()      // Engine A
  ELSE IF H1 ADX >= 15:
    signal = tryFvgFillEntry()          // Engine B
  ELSE IF H1 ADX < 20 AND mode != DEFENSIVE:
    signal = tryRangeEntry()            // Engine C

  IF signal is null: SKIP

  // Pre-entry filters
  IF same-day SL count for this direction >= 3: SKIP
  IF direction stacking guards block: SKIP

  // Lot sizing
  qualityScore = calculateQualityScore(signal)
  slPoints = |entryPrice - slPrice|
  lotSize = riskManager.calculateLotSize(slPoints, qualityScore)

  // Position opening
  IF (STRONG_TREND OR qualityScore >= 65) AND lotSize >= 0.02:
    OPEN runner at full lotSize (normal TP + trail)
    OPEN bonus locker at 30% lotSize (TP at 1.0R)
  ELSE:
    OPEN single position at lotSize

  SET cooldown (1-2 candles)
```
